# 请假功能使用说明

## 1. 首次启用

如果数据库已经创建过，只需要在 MySQL 中执行项目根目录下的：

```sql
SOURCE leave_feature.sql;
```

也可以在 MySQL Workbench 中打开 `leave_feature.sql` 并执行。该脚本会创建
`leave_request` 请假申请表，不会删除已有签到数据。

全新部署时可直接执行上一级目录的 `attendance_system.sql`，其中已经包含请假表。

## 2. 启动服务

```powershell
cd code
pip install flask flask-cors pymysql
python app.py
```

另开一个终端启动统计服务：

```powershell
cd code
python query.py
```

核心业务服务使用 5000 端口，统计服务使用 8080 端口。

## 3. 学生申请请假

1. 以学生身份登录。
2. 进入“我的”，打开“我的签到历史”。
3. 在“请假申请”区域选择课程签到场次。
4. 填写请假原因并提交。
5. 在下方申请记录中查看“待审批、已批准、已驳回”状态。
6. 已驳回的场次可以修改原因后重新提交。

已经正常签到、迟到签到或已批准请假的场次不会再次出现在可申请列表中。

## 4. 教师审批请假

1. 以教师身份登录并进入“发起签到”页面。
2. 在“请假审批”区域查看本人教学班的申请。
3. 点击“批准”或“驳回”。驳回时必须填写原因。
4. 批准后，系统会生成 `leave` 考勤记录；如果该学生已被补录为缺勤，原缺勤记录会自动改成请假。
5. 学生已经完成有效签到时，系统会阻止重复批准请假。

## 5. 统计规则

- `present`：正常出勤，计入出勤率分子和分母。
- `late`：迟到，按有效出勤计入分子和分母。
- `absent`：缺勤，只计入分母。
- `invalid`：无效扫码，不作为有效出勤。
- `leave`：已批准请假，不计为缺勤，也不进入应到分母。

后台签到记录页面支持查询、显示和手动修改 `leave` 状态。

## 6. 请假接口

| 方法 | 接口 | 作用 |
| --- | --- | --- |
| GET | `/api/students/<student_id>/leave_options` | 查询可申请场次 |
| GET | `/api/students/<student_id>/leave_requests` | 查询学生申请记录 |
| POST | `/api/leave_requests` | 提交或重新提交申请 |
| GET | `/api/teacher/<teacher_id>/leave_requests` | 查询教师名下申请 |
| POST | `/api/leave_requests/<request_id>/review` | 批准或驳回申请 |
