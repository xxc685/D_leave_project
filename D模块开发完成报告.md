# D模块开发完成报告 —— 出勤大数据分析

---

## 一、任务概述

负责整个系统的**出勤数据分析与可视化**模块，包括：

- **统计后端服务**（独立服务，端口 8080）
- **Web管理端统计面板**（Vue 3 + ECharts）
- **小程序统计看板**（学生/教师角色区分）

实现从数据采集、计算到前端展示的完整链路，支持多维度出勤分析、异常预警和个性化查询。

---

## 二、完成的业务功能

### 1. 统计后端服务（`query.py`）

独立运行于 8080 端口，提供 13 个 RESTful API，涵盖以下维度：

| 接口 | 功能 | 说明 |
|------|------|------|
| `/api/statistics/student/<id>` | 学生个人出勤汇总 | 总场次、出勤/迟到/缺勤次数、出勤率 |
| `/api/statistics/student/<id>/records` | 学生签到明细记录 | 按时间倒序返回所有签到记录 |
| `/api/statistics/class/<id>/overview` | 教学班整体概览 | 出勤率、场次数、选课人数、缺勤总数 |
| `/api/statistics/class/<id>/trend` | 教学班各场次趋势 | 每场出勤率变化折线数据 |
| `/api/statistics/class/<id>/absent_students` | 班级缺勤学生名单 | 按缺勤次数降序排列 |
| `/api/statistics/course/<course_id>/classes` | 课程下各教学班对比 | 同一课程不同班级出勤率对比 |
| `/api/statistics/department` | 院系统计 | 各院系平均出勤率及学生人数 |
| `/api/statistics/teacher/<id>/summary` | 教师汇总 | 教师名下所有班级出勤率及明细 |
| `/api/statistics/trend/daily` | 每日出勤趋势 | 全校每日出勤率变化 |
| `/api/statistics/warning/continuous_absent` | 连续缺勤预警 | 缺勤≥3次的学生列表 |
| `/api/statistics/warning/invalid_signin` | 无效签到统计 | 无效签到次数最多的学生 |
| `/api/statistics/session/<id>/absent_list` | 场次未签到名单 | 某场次已选课但未签到的学生 |
| `/api/teachers` | 教师列表（供下拉使用） | 所有教师工号和姓名 |

> 所有接口均采用统一返回格式 `{ code, msg, data }`，支持 CORS 跨域。

### 2. Web 管理端统计面板（`admin_web/index.html`）

在原有管理端中新增“出勤大数据分析”菜单，包含六个选项卡：

| 选项卡 | 功能 | 呈现方式 |
|--------|------|----------|
| **综合看板** | 各院系出勤率柱状图 + 缺勤TOP10横向柱状图 | ECharts |
| **个人查询** | 输入学号，展示总出勤率、按课程分类统计、缺勤明细、签到记录表 | 表格 + 进度条 |
| **教学班统计** | 选择教学班，展示概览数字、出勤率趋势折线图、同课程班级对比柱状图、缺勤学生名单 | 数字 + ECharts + 表格 |
| **教师统计** | 选择教师，展示各教学班出勤率柱状图 | ECharts |
| **时间趋势** | 选择日期范围，展示每日出勤率折线图 | ECharts + 日期选择器 |
| **异常预警** | 连续缺勤列表 + 无效签到统计 + 场次未签到学生名单 | 表格 |

**技术实现**：
- Vue 3 + Element Plus + Axios（CDN 引入）
- ECharts 5 图表渲染
- 综合看板每日 0 点自动刷新
- 窗口自适应 resize，所有图表在切换选项卡时正确调整尺寸

### 3. 小程序统计看板（`pages/stats/`）

根据用户角色（学生/教师）展示不同内容：

- **学生端**：
  - 总出勤率 + 总签到次数卡片
  - 按课程分组的出勤明细（课程名称、出勤率、有效/总次数）
  - 缺勤课程及日期列表

- **教师端**：
  - 下拉选择所带班级
  - 当前班级出勤率、场次数、选课人数
  - 缺勤学生名单（按次数降序）
  - 所有班级出勤率概览列表

**数据来源**：调用统计后端（`:8080`）接口，实时渲染。
