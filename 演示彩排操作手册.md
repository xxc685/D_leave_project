# 演示彩排操作手册

本手册用于从零开始完整演示一次“教师发起签到、学生签到、学生请假、教师审批、结束场次、查看统计”的业务流程。

## 一、固定演示账号

| 用途 | 身份 | 登录 ID | 姓名 |
| --- | --- | --- | --- |
| 创建签到和审批 | 教师 | `T001` | 张三 |
| 正常扫码 | 学生 S1 | `2024001` | 陈志远 |
| 申请请假 | 学生 S2 | `2024030` | 邱思敏 |

演示课程为“数据库原理”，教学班为 `CL002`，地点为 教3-203。

## 二、第一次运行前的准备

### 第 1 步：修改数据库密码

打开：

```text
D_leave_project/code/db.py
```

把 `password` 修改为本机 MySQL 的 `root` 密码。保存后不要把密码展示在汇报 PPT 或截图中。

### 第 2 步：准备数据库

打开 MySQL Workbench 并连接本机 MySQL。

如果电脑上还没有 `attendance_system` 数据库：

1. 打开 `D_leave_project/attendance_system.sql`。
2. 点击闪电按钮执行全部 SQL。
3. 确认左侧 Schemas 中出现 `attendance_system`。

如果数据库已经存在：

1. 打开 `D_leave_project/leave_feature.sql`。
2. 执行全部 SQL。
3. 确认 Tables 中出现 `leave_request`。

### 第 3 步：导入固定演示数据

1. 在 MySQL Workbench 中打开 `D_leave_project/demo_data.sql`。
2. 执行全部 SQL。
3. 查看底部结果，应出现：
   - 教师 `T001`；
   - 教学班 `CL002`；
   - 学生 `2024001` 和 `2024030` 的两条在选记录。

`demo_data.sql` 可以重复执行，不会重复创建账号。

### 第 4 步：安装 Python 依赖

打开 PowerShell：

```powershell
cd "C:\数据库及实现\课程pj\D_leave_project\code"
python -m pip install flask flask-cors pymysql
```

只需成功安装一次。

### 第 5 步：生成历史统计数据

确认数据库连接密码正确后，在 PowerShell 中运行：

```powershell
cd "C:\数据库及实现\课程pj\D_leave_project\code"
python demo_simulate.py
```

该脚本会为 CL001 和 CL002 各生成 5 周历史场次。**它会先清空 `leave_request`、`attendance_record` 和 `attendance_session`，只能用于演示数据库。**

## 三、每次演示前启动系统

### 第 1 个终端：核心业务服务

```powershell
cd "C:\数据库及实现\课程pj\D_leave_project\code"
python app.py
```

看到下面的信息即为成功：

```text
Running on http://127.0.0.1:5000
```

不要关闭这个终端。

### 第 2 个终端：统计服务

```powershell
cd "C:\数据库及实现\课程pj\D_leave_project\code"
python query.py
```

看到下面的信息即为成功：

```text
Running on http://127.0.0.1:8080
```

不要关闭这个终端。

### 浏览器检查

依次打开：

```text
http://127.0.0.1:5000/api/teacher/T001/classes
http://127.0.0.1:8080/
```

第一个地址应返回 `数据库原理`，第二个地址应返回统计接口列表。

如果第一个地址出现 `Access denied`，说明 `db.py` 密码不正确；如果出现 `Unknown table leave_request`，说明没有执行 `leave_feature.sql`。

## 五、单窗口完整彩排

单窗口彩排需要反复退出和登录，适合现在检查所有功能。正式汇报建议使用多个独立窗口提前登录。

### 阶段 A：教师创建签到

1. 登录页选择“教师登录”。
2. 工号输入 `T001`。
3. 姓名输入“张三”。
4. 点击“登录”。
5. 首页点击“发起签到”。
6. 确认班级选择器显示“数据库原理 · 教3-203”。
7. 签到有效时长输入 `2` 分钟。
8. 保持“开启地理位置校验”为关闭状态。
9. 点击“立即发起签到”。
10. 确认页面显示二维码、签到口令和场次编号。
11. 使用 Windows 截图工具截取二维码，并保存为 `demo_qr.png`。
12. 记下页面显示的场次编号，例如“场次 #12”。后续申请请假时要选择最新场次。

预期结果：教师端弹出“签到发起成功”，历史签到记录顶部出现一个“进行中”场次。

### 阶段 B：学生 S1 正常签到

1. 进入底部“我的”。
2. 点击“退出登录”并确认。
3. 登录页选择“学生登录”。
4. 学号输入 `2024001`。
5. 姓名输入“陈志远”。
6. 点击“登录”。
7. 首页点击“扫一扫签到”。
8. 在开发者工具扫码窗口中选择“从图片识别”，选择刚保存的 `demo_qr.png`。
9. 第一次使用时允许获取位置信息。即使教师关闭定位校验，学生端仍会读取当前位置。
10. 等待“签到结果”弹窗。
11. 点击“查看记录”。
12. 在“我的签到历史”中确认最新记录为：
    - 课程：数据库原理；
    - 状态：出勤；
    - 日期：当天日期。

预期结果：提示“签到成功：已出勤”。重复扫描同一二维码时应提示已经签到过。

### 阶段 C：学生 S2 提交请假

1. 进入底部“我的”。
2. 点击“退出登录”。
3. 选择“学生登录”。
4. 学号输入 `2024030`。
5. 姓名输入“邱思敏”。
6. 登录后进入底部“我的”。
7. 点击“我的签到历史”。
8. 找到“请假申请”区域。
9. 点击场次选择器。
10. 选择“数据库原理”且场次编号与阶段 A 一致的最新场次。
11. 请假原因输入：

```text
身体不适，申请本次课程请假。
```

12. 点击“提交请假申请”。
13. 在下方“申请记录”确认状态为“待审批”。

预期结果：该场次从可申请列表中消失，申请记录中出现一条待审批记录。

### 阶段 D：教师批准请假

1. 返回“我的”并退出学生登录。
2. 选择“教师登录”。
3. 工号输入 `T001`，姓名输入“张三”。
4. 登录后点击首页“发起签到”。
5. 向下滚动到“请假审批”。
6. 点击“刷新”。
7. 找到“邱思敏（2024030）”的申请。
8. 检查课程、日期、场次编号和请假原因。
9. 点击“批准”。
10. 审批备注输入：

```text
情况属实，同意请假。
```

11. 再次确认批准。

预期结果：申请状态从“待审批”变为“已批准”。后端同时生成一条 `leave` 考勤记录。

### 阶段 E：学生验证审批结果

1. 退出教师登录。
2. 使用 `2024030` 和“邱思敏”重新登录。
3. 进入“我的”“我的签到历史”。
4. 点击请假区域的“刷新”。
5. 确认申请状态为“已批准”。
6. 确认审批备注显示“情况属实，同意请假”。
7. 向下查看签到历史。
8. 确认最新考勤记录状态为“请假”，备注中包含请假原因。
9. 返回“我的”，打开“出勤统计看板”。
10. 确认“已批准请假”为 1，请假没有出现在缺勤记录中。

### 阶段 F：教师关闭签到场次

此时距离创建场次通常已经超过 2 分钟。

1. 退出学生登录。
2. 使用 `T001` 重新登录教师端。
3. 点击“发起签到”。
4. 在“历史签到记录”中找到刚才的“进行中”场次。
5. 点击该场次，页面会重新载入二维码和场次编号。
6. 点击“结束本次签到”。
7. 阅读确认提示并点击“确定结束”。

预期结果：

- 场次变为“已结束”；
- 缺勤补录人数为 28；
- 2024001 已正常签到；
- 2024030 已批准请假，因此不会被补录为缺勤；
- CL002 其余 28 名学生未操作，系统会将其补录为缺勤。

如果提示“尚未到 end_time”，等待到创建场次满 2 分钟后再次操作。

### 阶段 G：查看教师统计

1. 进入底部“我的”。
2. 点击“出勤统计看板”。
3. 选择 `CL002`。
4. 检查场次数、选课人数和出勤率。
5. 缺勤名单中不应出现 2024030。

本场 CL002 共 30 名在选学生，2024030 的批准请假从应到人数中排除；2024001 正常签到，其余学生会被补录为缺勤。班级总体统计还包含 `demo_simulate.py` 生成的 5 周历史数据，因此以页面实际汇总值为准，并重点确认 2024030 不在缺勤名单中。

## 八、常见错误排查

### 教师端显示“暂无授课班级”

- 检查工号是否为 `T001`。
- 重新执行 `demo_data.sql`。
- 浏览器打开 `/api/teacher/T001/classes` 检查返回结果。

### 学生端没有可申请场次

- 检查学生是否为 `2024030`。
- 检查教师是否已经创建新场次。
- 检查该学生是否已经签到、已提交申请或已有批准请假。
- 点击请假区域“刷新”。

### 提示 `Table leave_request doesn't exist`

执行 `leave_feature.sql` 后重启 `app.py`。

### 提示网络请求失败

- 检查 `app.py` 是否在 5000 端口运行。
- 检查 `query.py` 是否在 8080 端口运行。
- 检查 `utils/config.js` 是否仍为 `127.0.0.1`。
- 真机调试时改为电脑局域网 IP。

### 数据库提示 1045 Access denied

`code/db.py` 中的 MySQL 用户名或密码错误。修改后重新启动两个 Python 服务。

### 无法结束签到

教师端默认不允许在结束时间前关闭。等待设置的有效时长结束后重试。

### 二维码无法从图片识别

- 重新截取完整二维码，确保四周留白。
- 使用教师页面显示的 `qr_token` 作为开发者工具模拟扫码结果。
- 不要使用上一轮彩排生成的旧二维码。
